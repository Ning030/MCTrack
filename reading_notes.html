一、项目全貌

1.1 项目简介

MCTrack（Multi-Camera Track）是一个统一的三维多目标追踪（3D MOT）框架，面向自动驾驶场景，已被 IROS 2025 接收。

解决的问题：现有跟踪方法往往针对单一数据集设计，跨数据集泛化能力差。MCTrack 提出统一的 BaseVersion 数据格式和通用追踪管线，在 KITTI、nuScenes、Waymo 三大基准上同时达到 SOTA。

核心亮点：

  • 统一的 BaseVersion 数据格式，消除数据集差异
  • BEV + RV 双阶段匹配策略
  • 支持 CV / CA / CTRA 三种运动模型的扩展卡尔曼滤波
  • 创新的运动指标评估体系（VAE / VNE / VDE 等）
  • 支持在线（ONLINE）和全局（GLOBAL）两种追踪模式

1.4 快速复现

① 环境安装

# 创建 conda 环境
conda create -n MCTrack python=3.8
conda activate MCTrack

# 安装依赖（requirements.txt:1-9）
pip install -r requirements.txt
# 核心依赖: PyYAML, lap, easydict, motmetrics==1.1.3,
#           numba==0.58.1, numpy==1.22.0, pandas==1.5.0,
#           pillow==10.4.0, nuscenes-devkit==1.1.11

② 数据准备

# 步骤 1: 下载原始数据集 + 检测结果
# 步骤 2: 转换为 BaseVersion 格式
python preprocess/convert2baseversion.py --dataset kitti
# 输出: data/base_version/kitti/{detector}/{split}.json

③ 运行追踪 + 评估

# 单进程运行
python main.py --dataset kitti -e -p 1

# 多进程运行（8 workers）
python main.py --dataset kitti -e -p 8

# nuScenes / Waymo
python main.py --dataset nuscenes -e -p 8
python main.py --dataset waymo -e -p 8

# 离线模式
python main.py --dataset kitti -e -m

# 调试模式（仅处理 2 个场景）
python main.py --dataset kitti --debug

核心方法

predict() — :168-184

  • 依次调用 pose/yaw/size 三个卡尔曼滤波器的 predict()
  • 将预测结果写入 bboxes[-1].global_xyz_lwh_yaw_predict

update(bbox, matched_score) — :186-241

  • 输入：匹配到的检测 BBox + 匹配代价分数
  • 流程：
    1. 将新 bbox 追加到 bboxes，超过 _cache_bbox_len 则 pop(0)
    2. 计算差分速度 cal_diff_velocity() 和曲线拟合速度 cal_curve_velocity()
    3. 用新观测更新三个滤波器 (pose → yaw → size)
    4. 融合后的状态写入 global_xyz_lwh_yaw_fusion
    5. 根据追踪长度或匹配/检测分数，判断是否将 status_flag 设为 1（已确认）

unmatch_update(frame_id) — :243-290

  • 未被匹配时调用，执行"假更新"：用预测值作为观测值喂给滤波器
  • 生成 fake_bbox（det_score=0, is_fake=True）填充轨迹
  • 状态转移逻辑：
    • status_flag 0 → 4：初始化轨迹超过确认长度仍未确认 → 死亡
    • status_flag 1 → 2：已确认但连续未匹配超过 _max_unmatch_len → 遮挡
    • status_flag 2 → 4：遮挡且连续未匹配超过 _max_predict_len → 死亡

cal_diff_velocity() — :349-361

  • 用相邻两帧的位置差 / 时间差计算速度

cal_curve_velocity() — :363-378

  • 对最近 3 帧做线性拟合 (scipy.optimize.curve_fit)，取斜率作为速度估计

filtering() — :299-347

  • 全局模式后处理：对未匹配的中间帧做线性插值，平滑评分
  • 参考自 3D-Multi-Object-Tracker

来源：tracker/trajectory.py:18-378

3.5 匹配算法详解

文件：tracker/matching.py:1-211

match_trajs_and_dets()

:107-160 — 统一匹配入口

  • 输入：轨迹列表 trajs, 检测列表 dets, 配置 cfg, 可选变换矩阵, is_rv 标志
  • 输出：matched_indices (N×2 数组, 列0=轨迹索引, 列1=检测索引), costs
  • 流程：
    1. 调用 cost_calculate_general() 生成代价矩阵
    2. 用 mask_tras_dets() 构建类别掩码，异类对设为 inf
    3. 根据 MATCHING_MODE 调用 Hungarian 或 Greedy

Hungarian()

:61-104 — 匈牙利算法

  • 内部调用 lap.lapjv(cls_cost, extend_cost=True, cost_limit=threshold)
  • extend_cost=True：自动扩展为方阵以处理不等数量的轨迹/检测
  • cost_limit：超过阈值的匹配被拒绝
  • 适用场景：最优匹配，结果全局最优

Greedy()

:12-58 — 贪心算法

  • 遍历每个检测，选择代价最小的轨迹进行匹配
  • 匹配后将该轨迹列设为 1e18（防止重复匹配）
  • 适用场景：实时性要求高的场景，复杂度 O(N²) vs Hungarian O(N³)

cost_calculate_general()

:163-211 — 代价矩阵生成

  • 支持 Predict / BackPredict / Fusion 三种代价状态
  • Predict：用轨迹的卡尔曼预测位置 vs 检测当前位置
  • BackPredict：用轨迹的融合位置 vs 检测的反向预测位置
  • Fusion：按 COST_STATE_PREDICT_RATION 加权混合两者
  • 最终 cost_matrix = 1 - similarity_matrix，类别不匹配设为 inf

3.6 代价函数详解

文件：tracker/cost_function.py:1-253

cal_rotation_gdiou_inbev() — RO-GDIoU 3D

:34-97 — BEV 阶段的核心代价函数

  • 输入：box_trk (Trajectory), box_det (BBox), cfg, cal_flag
  • 输出：ro_gdiou (float) — 旋转广义 IoU 分数
  • 算法步骤：
    1. 根据 cal_flag 取对应的 pose（Predict 或 Fusion）
    2. 调用 transform_3dbox2corners() 生成 8 个 3D 角点
    3. 提取 BEV 角点（索引 [2,3,7,6]），调用 cal_rotation_iou_inbev() 计算旋转 IoU
    4. 用 cv2.minAreaRect() 计算最小包围矩形
    5. 计算额外面积比 (extra_area / universe_area)、中心距离比 (center_dist / union_dist)
    6. 公式: ro_gdiou = iou - w1 × (extra_area / universe_area) - w2 × (center_dist² / union_dist²)
  • 权重 w1, w2 来自 cfg["THRESHOLD"]["BEV"]["WEIGHT_RO_GDIOU"]

cal_sdiou_inrv() — SDIoU 2D

:205-252 — RV 阶段的核心代价函数

  • 仅处理 CAM_FRONT 相机的检测
  • 公式：sdiou = iou + area_ratio × distance_ratio × aspect_ratio
  • 综合考虑面积比、中心距比、长宽比三个维度

cal_rotation_iou_inbev()

:12-31 — BEV 旋转 IoU 计算

  • 使用 cv2.rotatedRectangleIntersection() 计算两个旋转矩形的交集
  • 用 cv2.convexHull() + cv2.contourArea() 计算交集面积

EKF_CV — 匀速模型

extend_kalman.py:147-213

  • 状态向量：[x, y, vx, vy] (n=4)
  • 观测向量：[x, y] 或 [x, y, vx, vy] (m=2 或 4)
  • 状态转移：x' = x + dt·vx, y' = y + dt·vy
  • 雅可比矩阵 F：
F = | 1  0  dt  0  |
    | 0  1  0   dt |
    | 0  0  1   0  |
    | 0  0  0   1  |

EKF_CA — 匀加速模型

extend_kalman.py:215-287

  • 状态向量：[x, y, vx, vy, ax, ay] (n=6)
  • 状态转移：x' = x + dt·vx + 0.5·dt²·ax, vx' = vx + dt·ax

EKF_CTRA — 恒转弯率加速模型

extend_kalman.py:289-450

  • 状态向量：[x, y, yaw, v, ω, a] (n=6)
  • 状态转移（非线性）：
x' = x + (1/ω²)·[(v·ω + a·ω·dt)·sin(yaw+ω·dt) + a·cos(yaw+ω·dt)
                   - v·ω·sin(yaw) - a·cos(yaw)]
y' = y + (1/ω²)·[(-v·ω - a·ω·dt)·cos(yaw+ω·dt) + a·sin(yaw+ω·dt)
                   + v·ω·cos(yaw) - a·sin(yaw)]
yaw' = yaw + ω·dt
v' = v + a·dt
  • 雅可比矩阵 F 为 6×6 矩阵，各元素是状态转移函数的偏导数（:330-423）
  • 适用场景：弯道行驶等需要考虑转弯率的复杂运动

  • 特殊处理：弧度归一化 norm_radian() 避免角度回绕问题
