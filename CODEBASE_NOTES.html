MCTrack — 代码阅读笔记

一句话定位：MCTrack 是一个基于运动中心（Motion-Centric）的 3D 多目标跟踪框架，在 KITTI、nuScenes、Waymo 三大基准上取得 SOTA 性能，已被 IROS 2025 收录。

摘要（≤300 字）： MCTrack 采用 Python 实现，以 YAML 驱动配置。核心流程为：将原始检测结果预处理为统一的 BaseVersion 格式 → 通过 main.py 逐场景、逐帧执行两阶段匹配（BEV 主匹配 + RV 辅助匹配）→ 每条轨迹由 5 个独立 Kalman 滤波器（CV/CA/CTRA/Yaw/Size）维护状态 → 支持在线/离线模式以及轨迹插值平滑 → 使用官方评估工具计算 HOTA/AMOTA/MOTA 等指标，同时提出了速度/加速度运动指标评估。主要依赖 NumPy、lap（线性分配）、numba（JIT）、nuscenes-devkit 和 PyYAML。仓库约 170 个文件，纯 Python，入口为 main.py（跟踪）和 preprocess/convert2baseversion.py（数据预处理）。

Layer 1 — 项目是什么 & 怎么组织的

README / Docs 摘要

  • MCTrack 是一个 Motion-Centric 3D 多目标跟踪方法，利用运动信息（速度、加速度）进行关联与状态估计。
  • 在 KITTI（HOTA 82.75）、nuScenes（AMOTA 0.763）、Waymo（MOTA L1 0.7504）三大基准取得 SOTA 或接近 SOTA 性能。
  • 已被 IROS 2025 接收，2024 年 10 月开源。
  • 提供在线（逐帧）与离线（全序列优化）两种跟踪模式。
  • 创新性地引入了面向下游任务的运动指标评估（速度、加速度精度）。

标识

名称cost_calculate_general(trajs, dets, cfg, mode)
文件tracker/matching.py
行范围L163 – L211

输入 / 输出

类型内容
Input轨迹列表、检测列表、配置、模式（BEV/RV）
Outputcost_matrix: np.ndarray [n_tra × n_det]，值域 [0, 1]（0=完美匹配）

主要实现逻辑

  • 遍历所有轨迹-检测对，根据模式选择代价函数：
    • BEV 模式: RO_GDIOU_3D → 调用 cal_rotation_gdiou_inbev()
    • RV 模式: IOU_2D / GIOU_2D / DIOU_2D / SDIOU_2D → 对应 RV 函数
  • 计算 Predict 代价（用预测框 vs 检测框）和 BackPredict 代价（用上一帧反向预测）
  • 根据配置融合：Predict only / BackPredict only / Fusion（加权平均）
  • 最终代价 = 1 - similarity（转为最小化问题）
依赖cost_function 模块中的 5 个距离函数
测试无独立单元测试
性能O(n_tra × n_det) 复杂度，cal_rotation_gdiou_inbev() 每次调用涉及 OpenCV 旋转矩形求交

节点 6：EKF 运动模型（CV / CA / CTRA）

标识

名称EKF_CV, EKF_CA, EKF_CTRA
文件kalmanfilter/extend_kalman.py
行范围EKF_CV: L147–L212；EKF_CA: L215–L286；EKF_CTRA: L289–L449

输入 / 输出

主要实现逻辑

  • EKF_CV（匀速）：最简模型，线性 F 矩阵，适用于直线运动
  • EKF_CA（匀加速）：在 CV 基础上加入加速度状态，6D 线性模型
  • EKF_CTRA（匀转弯率+加速度）：非线性模型，Jacobian 包含三角函数，适用于弯道场景
  • CTRA 的 f(x) 中处理了 turn_rate ≈ 0 的退化情况（L310-330）
  • 所有模型继承 KF_Base，使用统一的 predict-update 框架

关键代码片段

# kalmanfilter/extend_kalman.py L310-330 (CTRA 状态转移)
def f(self, x):
    if abs(x[4]) > 0.001:  # turn_rate 非零
        x_new[0] = x[0] + (x[3]/x[4]) * (sin(x[2]+x[4]*dt) - sin(x[2]))
        x_new[1] = x[1] + (x[3]/x[4]) * (cos(x[2]) - cos(x[2]+x[4]*dt))
    else:  # 近似直线
        x_new[0] = x[0] + x[3] * cos(x[2]) * dt
        x_new[1] = x[1] + x[3] * sin(x[2]) * dt
依赖KF_Base (base_kalman.py), numpy, utils.norm_radian
测试无独立单元测试
性能CTRA Jacobian 每次 predict 需要计算多个三角函数，但对象数量有限，非瓶颈
风险CTRA 在极低速度 + 高转弯率时可能数值不稳定

建议下一步：查看 KF_YAW 的角度归一化处理（norm_radian）。
